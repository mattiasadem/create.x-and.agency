## Core Principles

### 1) CLARITY OVER CLEVERNESS
- Say what you do in plain language
- Benefits before features
- One message per section

### 2) SOCIAL PROOF IS KING
- Numbers, logos, testimonials
- Specific results beat vague claims
- Show, don't tell

### 3) TEST EVERYTHING
- Hypotheses, not opinions
- A/B test headlines, CTAs, layouts
- Learn from data, iterate fast

## Positioning Framework

### Positioning Statement Template
```
For [TARGET CUSTOMER]
who [PROBLEM/NEED],
[PRODUCT NAME] is a [CATEGORY]
that [KEY BENEFIT].
Unlike [ALTERNATIVE],
we [DIFFERENTIATOR].
```

### Example
```
For solo founders
who want to ship products fast,
x-and is an AI product creation platform
that turns business intent into shipped products.
Unlike Bolt or v0,
we build apps, brands, and automations — all connected.
```

## Landing Page Structure

### Hero Section
```markdown
## [Headline: Big Promise]
[Subhead: How you deliver on promise, 1 sentence]

[Primary CTA: "Get Started Free"]
[Secondary CTA: "See Demo"]

[Hero image/video showing the product]
```

### Problem Section
```markdown
## [Problem headline]
- Pain point 1
- Pain point 2
- Pain point 3

"Sound familiar?"
```

### Solution Section
```markdown
## [How it works headline]

1. [Step 1 with visual]
2. [Step 2 with visual]
3. [Step 3 with visual]
```

### Social Proof Section
```markdown
## [Trust headline]

[Logos of customers/featured in]

[3 testimonials with photo, name, title, quote]

[Key metrics: "10,000+ products launched"]
```

### Features Section
```markdown
## [Features headline]

[Feature cards with icon, title, description]
- Feature 1
- Feature 2
- Feature 3
```

### Pricing Section
```markdown
## [Pricing headline]

[Pricing tiers with clear differentiation]
[Most popular tier highlighted]
[FAQ or objection handling below]
```

### Final CTA Section
```markdown
## [Final push headline]

[Recap of value prop]
[Primary CTA]
[Risk reversal: "Free to start, no credit card"]
```

## Copy Formulas

### Headlines
```
- [Do X] without [Pain Y]
- The [Benefit] of [Category], without the [Downside]
- [Number] ways to [Benefit] in [Timeframe]
- Stop [Pain]. Start [Benefit].
```

### CTAs
```
Strong: "Start building free" / "Get started now" / "Try it free"
Medium: "Learn more" / "See how it works"
Weak: "Submit" / "Click here" ❌
```

### Value Props
```
- Feature → Benefit → Result
- "AI-powered code generation" → "Build faster" → "Launch in days, not months"
```

## SEO Essentials

### On-Page
```html
<title>Primary Keyword - Brand | Secondary Keyword</title>
<meta name="description" content="Compelling 155 chars with keywords and CTA">
<h1>Main Keyword in Natural Headline</h1>
```

### Technical
- Fast page load (< 3s)
- Mobile responsive
- Proper heading hierarchy (H1 → H2 → H3)
- Alt text on images
- Internal linking

### Content Strategy
```
1. Identify target keywords (search volume + intent match)
2. Create pillar content (comprehensive guides)
3. Create cluster content (specific topics linking to pillar)
4. Update content quarterly
```

## Funnel Metrics

| Stage | Metric | Good Benchmark |
|-------|--------|----------------|
| Awareness | Traffic, impressions | Growing MoM |
| Interest | Time on page, scroll depth | > 60s, > 50% |
| Desire | Email signups, demo requests | 3-5% of traffic |
| Action | Trial starts, purchases | 20-30% of leads |

## A/B Test Ideas

### High Impact
- Headline variations
- CTA button text and color
- Hero image vs video
- Pricing presentation
- Social proof placement

### Low Impact (avoid)
- Font sizes
- Minor color tweaks
- Footer changes
